# 设计评审报告

> 评分采用 `huashu-design` 5 维框架（各 0–10）：**哲学一致性 / 视觉层级 / 细节执行 / 功能性 / 创新性**。
> 严重程度标记：⚠️ 致命（破坏可用性或专业感）· ⚡ 重要（明显影响体验）· 💡 优化（精致度）。

---

## 一、执行摘要

项目是一个 Vue3 + Element Plus 个人博客，结构完整、功能可用、昼夜主题机制优雅。但从「生产级 / 消费级」标准看，**核心短板高度集中且高度一致**，可归纳为四条主线：

1. **没有真正的设计系统**。主题变量按「组件」散装命名（如 `--english_left_funtion_item_hover`），缺排版尺度、间距尺度、阴影/层级、动效、语义色。全站存在 **17 个互不相关的强调色**、**15+ 种字号**、**8+ 种圆角**，无法一句话说清品牌主色。（详见 [02-token-audit.md](./02-token-audit.md)）

2. **占位内容混入生产页面**。Home 文章卡是硬编码 `v-for="movie in 10"` 的假数据；WebNavigate/WebSite 卡片是 Element 示例图与「Card name」占位；About 页几乎空白只有打字机 demo；Test 页是调试残留。**用户当前看到的是半成品。**

3. **两套设计语言并存**。博客主体（暖色纸感 + Element 卡片）与 AiEnglish 子应用（深色 ChatGPT 范式 + 高对比滤镜标题）像两个产品。需要在「统一」与「子应用独立气质」之间做出明确取舍。

4. **细节执行粗糙**。硬编码色（评论框 `2px solid green`、输入框 `#333ce1`、Footer `#165dff`）、长文行宽失控、代码块对比过强、聊天气泡按钮重叠、`flex-wrap: 1` 无效值等，单点不致命，累积起来拉低专业感。

> **结论**：必要性强、可行性高。建议路径 = 先收敛设计令牌 + 清理占位 →（选定方向后）统一视觉语言 → 精修细节。

## 三、逐页评审

### 3.1 Home 首页 — 均分 5.6

**现状**：左栏（新闻轮播 + 文章列表 + 分页）+ 右栏（广告/作者/推荐/打赏 4 卡）。大屏 `el-row` 栅格、小屏单列。文章卡有 hover 缩放与标题下划线渐变动效。

**Keep**
- 标题 hover 的渐变下划线（`background-size` 过渡）是有品味的小交互。
- 大屏/小屏双布局 + 右栏条件显隐，响应式骨架合理。
- 新闻轮播卡片的 active/deactivate 透明度分层有层次。

**Fix**
- ⚠️ **文章列表是硬编码假数据**（`Home.vue:114` `v-for="movie in 10"`，标题/摘要/图片全写死，图片是 Element CDN 汉堡占位图）。当前 → 生产页展示占位符；修复 → 接入真实文章数据源，卡片改为接收 props 的列表项组件。
- ⚡ **元信息图标三色随意**（`Home.vue:424/428/432` 日期=红、浏览=绿、点赞=黄）。当前 → 三个无关纯色并列；为什么 → 视觉噪音、无语义系统；修复 → 统一为中性灰 `--color-text-tertiary`，仅「点赞激活」用主色/danger。
- ⚡ **无间距系统**：`margin` 出现 5/10/15/20/25/30/50px 混用。修复 → 收敛到 8pt 网格（见 03 号规范）。
- 💡 **阴影硬编码** `box-shadow: 1px 2px 2px 2px #b8b2b3`（`Home.vue:443`）与变量版并存。修复 → 统一走 `--shadow-*` 令牌。
- 💡 **`el-divider__text` 背景写死 `#f0eeee`**（`Home.vue:512`），暗色下会突兀。修复 → 改 `var(--color-bg-default)`。

**Quick Wins**：① 文章卡接真实数据；② 元信息图标统一中性灰；③ 间距收敛到 8pt。

---

### 3.2 Article 文章页 — 均分 5.8

**现状**：三栏（正文 + 右侧作者/TOC/工具）。Markdown 经 marked + highlight.js 渲染，自定义 `zibuyu-markdown.css`。支持图片放大、点赞、返回顶/底、TOC 跳转。

**Keep**
- TOC 生成 + 锚点跳转稳定；图片点击放大 + 遮罩交互完善；代码高亮覆盖全面；暗/亮主题双支持。

**Fix**
- ⚠️ **正文行宽（measure）失控**：`.markdown-body { padding: 3% }` + `el-col span=11` 无 `max-width`，长文行宽易 > 80 字符。修复 → 正文容器 `max-width: 70ch`（约 720–760px），居中。
- ⚠️ **代码块对比过强 / 配色割裂**：Light 主题代码块底色 `rgb(33,7,7)` 近纯黑、暗红棕 `#381609` 与高亮色对比不足。修复 → Light 用 `#1e1e1e`/`#f6f8fa`，Dark 用 `#0d1117`，行内代码 Light/Dark 反差（黄 `#e3e70c` ↔ 蓝 `#2213f5`）统一为中性底 + 高对比字。
- ⚠️ **评论框 `border: 2px solid green`**（`Article.vue:356`）。修复 → `1px solid var(--color-border-default)`，focus 用主色。
- ⚡ **行高偏紧** `line-height: 1.5`（18px 正文）。修复 → 正文 `1.75`。
- ⚡ **TOC 无 active 高亮**（滚动时不跟随）。修复 → IntersectionObserver 高亮当前章节。
- ⚡ **链接全局 `:before` 加 ✍🏻 emoji**（`zibuyu-markdown.css:361`），污染 TOC 与正文链接。修复 → 移除。
- ⚡ **右栏固定态宽度跳动**（`395px` vs `480px`）。修复 → 统一宽度。
- 💡 h2/h3 彩色背景标签、blockquote `:after` 定位 bug（`bottom` 重复赋值）等执行瑕疵，随规范统一清理。

**Quick Wins**：① 正文 `max-width: 70ch` + 行高 1.75；② 代码块配色统一；③ 移除链接 emoji 前缀 + 评论框绿边。

### 3.3 About 关于页 — 均分 2.8（最低）

**现状**：几乎空白——一个 `about界面` 标题 + 一个输入框 + 打字机 demo，硬编码红底黑字，无响应式，无真实「关于」内容。

**Keep**：打字机交互思路、离开路由清定时器的内存意识可保留并重构。

**Fix**
- ⚠️ **内容空虚**：无个人简介/技能/经历/联系方式。这是个人博客最该表达品牌的页面，目前等于空页。修复 → 设计 Hero 自述 + 技能/标签 + 时间线 + 社交链接 + 联系方式。
- ⚠️ **全硬编码、脱离主题**：`background-color: red`、`color: black`，暗色下完全失效。修复 → 全部改语义令牌。
- ⚡ **无响应式**：固定像素盒。修复 → 栅格/flex + 断点。
- 💡 打字机若保留，应升级为 Hero 品牌宣言（大字号），否则删。

**Quick Wins**：① 填真实「关于」内容骨架；② 去硬编码改令牌；③ 打字机转正为 Hero 标语或移除。

---

### 3.4 WebNavigate 网址导航页（含 WebSite 卡片）— 均分 4.8

**现状**：搜索栏 + 推荐区（4 卡）+ 全部区（24 卡），`el-row/el-col` 栅格。`WebSite.vue` 卡片：圆形头像 + 标题 + 描述 + 底部「Card name」。

**Keep**：两层分区（推荐/全部）逻辑清晰；栅格响应式基础扎实；flex 居中分布。

**Fix**
- ⚠️ **占位数据 + 组件不可传参**：`WebSite.vue` 把 url/标题/描述写死在 script/template，24 张卡内容全相同，图是 Element 示例图。修复 → 改 props 化（`{ title, desc, url, favicon, category }`）+ v-for 真实数据 + 点击跳转。
- ⚠️ **无真实 favicon**：导航站核心就是 favicon + 名称，现用大头像无辨识度。修复 → 接 favicon 服务或本地图标。
- ⚡ **卡片无 hover / 无可点击暗示**。修复 → `hover: translateY(-4px) + shadow + cursor:pointer`。
- ⚡ **无分类标签**：推荐与全部之间只有分隔线无文字标题，24 卡平铺无分组。修复 → 加分区标题 + 按类目（开发/设计/社区…）分组。
- 💡 卡片底部「Card name」与顶部标题重复且无语义。修复 → 删除或改为类目标签/访问按钮。
- 💡 固定 `250×150px` 在小屏脆弱。修复 → `clamp()` 宽度 + 自适应高度。

**Quick Wins**：① 卡片 props 化接真实数据；② 加 favicon；③ 加 hover + 分区标题。

### 3.5 AiEnglish 子应用 — 均分 6.4

**现状**：左菜单（280px/折叠 40px）+ 主聊天区（`max-width: 900px`）+ 底部输入区；通用助手 + 口语教练双模块；流式输出、markdown 渲染、多 AI 配置持久化、语音入口。

**Keep**
- 输入交互成熟（Enter 发送 / Ctrl+Enter 换行、textarea 自动高度、语音入口）。
- AI 配置（URL/Key/Model/Temperature/Top-p/Max-tokens/History）增删改查 + localStorage 完整。
- 流式输出 + 中断保护 + 失败兜底；通用/口语双队列分离，利于扩展。

**Fix**
- ⚠️ **与博客是「两套设计语言」**：深色 ChatGPT 范式 + 左菜单 `filter: contrast(30)` 高对比标题，与博客暖色纸感割裂。修复 → 见 03/04，确立「同一产品 + 子应用变体」关系，复用令牌。
- ⚠️ **输入框边框硬编码** `#333ce1`/`#989cea`（`content/InputBar.vue:31/40`，该输入框被 AiEnglish 与多处复用），脱离主题。修复 → 改 `var(--english_input_area_border*)`。
- ⚡ **AI / 用户气泡对比不足**：Light 下 `#F7F7F7` vs `#E0EFFF` 差异约 3%。修复 → 拉开层次（见硬数据建议值）。
- ⚡ **消息操作按钮重叠**：`bottom: -18px` + 消息间距仅 8px。修复 → 间距 ≥ 28px 或改 `relative + margin`。
- ⚡ **缺空状态 / 加载骨架 / 错误重试设计**：仅一条硬编码欢迎语。修复 → 补 empty/loading/error 三态。
- ⚡ **设置弹框溢出视口** `max-height: 65vh` + 项多。修复 → 选项卡分组 + `width: min(90vw, 600px)`。
- 💡 气泡长文/代码横向溢出（`max-width: 75%` 未约束内部）；口语教练 `fill="skyblue"` SVG 波浪不适配主题；缺时间戳。

**Quick Wins**：① 输入框边框改令牌；② 修复消息按钮重叠；③ 补空/加载/错误三态。

### 3.6 Test 测试页 — 均分 6.6（应下线）

**现状**：英语单词浮动墙动画（`setInterval 100ms` 重绘 DOM）+ 左菜单 + 输入框，与 AiEnglish 风格一致。

**判断**：路由名 `test`、无导航入口、属调试/演示残留。**建议从生产路由移除**；若动画有价值，转正为 `ai_english` 的彩蛋/背景并重命名。

**Fix**
- ⚠️ 暴露在生产路由（`router/index.js`）→ SEO 污染 + 用户困惑。修复 → 移除路由或加 `import.meta.env.DEV` 守卫。
- ⚡ `flex-wrap: 1` 是无效 CSS（应为 `flex: 1`）。
- ⚡ 单词颜色 6 个硬编码纯色随机，脱离主题。
- 💡 `setInterval` 频繁重建 DOM → 改 `requestAnimationFrame` 或 canvas，省电更顺。

## 五、总优先级（Quick Wins · 全局 Top 5）

如果只做 5 件事，按 ROI：

1. **建立语义化设计令牌**（color/type/spacing/radius/shadow/motion）——所有后续优化的地基。
2. **收敛品牌主色 + 清理硬编码色**——一眼提升专业感与一致性。
3. **清除占位/假数据**（Home 文章卡、WebNavigate 卡片、About 内容）——从「半成品」到「真产品」。
4. **文章页排版三件套**：`max-width: 70ch` + 行高 1.75 + 代码块配色统一——直接改善核心阅读体验。
5. **下线 Test 页 + 修明显细节 bug**（`flex-wrap:1`、气泡按钮重叠、链接 emoji）。

> 详细分阶段计划见 [05-roadmap.md](./05-roadmap.md)。

# 设计令牌现状诊断

> 这份诊断回答一个问题：**为什么这个项目「看起来不够统一、不够专业」？** 答案不在某一个页面，而在缺少一套设计令牌（design tokens）系统。本文盘点现状、量化混乱、给出迁移思路。具体的目标令牌定义见 [03-design-spec.md](./03-design-spec.md)。

---

## 一、命名方式诊断：按「组件」散装，而非按「语义」

现有 `light.css` / `dark.css` 的变量全部是**按使用它的组件/位置**命名的：

```css
--header_background        /* 谁用？header。换个组件就得新建变量 */
--english_left_funtion_item_hover   /* 拼写错误 funtion，且强绑定到「英语页左菜单功能项」 */
--small_screen_menu_bg
--news_activate_background
--markdown_article_body_activated
```

| 命名风格 | 示例 | 问题 |
|---------|------|------|
| 按组件 | `--header_background`、`--english_left_bg` | 复用性差，新页面要新建变量 |
| 按组件+状态 | `--english_left_funtion_item_active` | 与上者混杂，无统一模式 |
| Markdown 专用 | `--zibuyu-second-title-gb` | 仅文章内有效，孤立 |
| 第三方 (GitHub) | `--color-prettylights-syntax-*`（100+） | 维护负担重 |
| Element Plus | `--el-bg-color` | 只有 2 条，未系统接入 |

**后果**：
- 同一个语义色（如「主蓝」）在不同文件各写一遍，改配色要同时改 4–5 处。
- 无法回答「这个站的品牌色是什么」。
- 新人不知道该用哪个变量，倾向于直接写死十六进制 → 硬编码越来越多。

## 二、量化混乱

### 2.1 强调色：17 个互不相关

> 这是最直观的「不专业」来源。一个成熟产品通常是 **1 主色 + 1 辅色 + 4 语义色 + 灰阶**。

**蓝色系（4 个，互不相关）**
| 色值 | 出处 | 用途 |
|------|------|------|
| `#0716ef` | `light.css:6` | Header 标语 |
| `#0A7AFF` | `light/dark.css` | 英语页 hover / input focus |
| `#3170a7` | `HeaderNavigate.vue:72` | 昼夜切换按钮 fill |
| `#86a7e5` | `dark.css:18` | 小屏菜单字体 |

**红色系（4 个）**
| 色值 | 出处 | 用途 |
|------|------|------|
| `#8b081c` | `light/dark.css` | 小屏菜单激活 |
| `#a93920` | `dark.css:15` | 新闻内容（暗色） |
| `#ef7060` | `zibuyu-markdown.css:22` | Markdown 二级标题 |
| `rgb(235,17,28)` | `light.css:42` | 英语菜单折叠图标 |

**其他强调色（9 个）**
`#165dff`（Footer 版本号）· `#adb4c5`（Footer）· `#8cd6f1`/`#6f4848`（菜单按钮激活 明/暗）· `#dd9898`/`#40406a`（英语左侧激活 明/暗）· `#ebefec`/`#484545`（英语右标题激活 明/暗）· `#630be7`（Markdown 链接）

外加硬编码在 `.vue` 中、未进变量的：评论框 `green`（`Article.vue:356`）、输入框 `#333ce1`/`#989cea`（`content/InputBar.vue`）、聊天 hover `#007aff`、口语教练 `skyblue` SVG、Test 页 6 个随机色 `#eccc68/#ffa502/#7bed9f/#a4b0be/#70a1ff/#ff6b81`。

### 2.2 字号：15+ 种，无尺度

实际出现：`12 / 13 / 14 / 15 / 16 / 17 / 18 / 20 / 22 / 25 / 26 / 28px`、`2em / 1.5em / 1.2em / 0.9em`，外加多种 `clamp()`。**无统一律动**，标题/正文对比时大时小。

### 2.3 间距：随意取值

实际出现：`5 / 8 / 10 / 12 / 15 / 16 / 18 / 20 / 30 / 40 / 50 / 100 / 200px`。**非 8 的倍数居多**，节奏混乱。

### 2.4 圆角：8 种混用

`5 / 10 / 12 / 15 / 20 / 25px / 30% / 1%`（`Article.vue` 图片 `border-radius: 1%` 是明显笔误级用法）。

### 2.5 阴影：几乎没有系统

全站基本只有一种 `box-shadow: 1px 2px 2px 2px var(--home_hover_shadow)`（且有硬编码 `#b8b2b3` 版本并存）。**无 elevation 层级**（卡片/弹层/导航应有不同高度的阴影）。

### 2.6 动效：散落

`100ms / 150ms(无) / 300ms / 500ms / 1s / 1.5s / 3s / 5s` 混用，无 fast/normal/slow 标准，无统一缓动函数。

## 三、缺失尺度盘点

| 尺度 | 现状 | 需要 |
|------|------|------|
| 排版 Type Scale | ❌ 无 | 一套有比例关系的字号阶梯（建议 ~1.25 模数） |
| 间距 Spacing Scale | ❌ 无 | 8pt 网格（4/8/12/16/24/32/48/64…） |
| 圆角 Radius | ⚠️ 混用 | 收敛到 4 档（sm/md/lg/full） |
| 阴影 Elevation | ❌ 仅 1 | 3–4 档（卡片/悬浮/弹层/模态） |
| 动效 Motion | ⚠️ 散落 | fast/normal/slow + 标准缓动 |
| 语义色 | ⚠️ 混乱 | primary + success/warning/danger/info |
| 中性灰阶 | ⚠️ 缺 | gray-50…gray-900（约 9–10 级） |
| 品牌主色 | ❓ 不明确 | 唯一确定（候选见 04 号方向文档） |
| 字体 Font | ⚠️ 半定义 | 明确 display / body / mono 三组 |

## 四、迁移思路：散装变量 → 三层语义令牌

> 目标不是「推倒重来」，而是**在现有 light/dark 双主题机制之上，插入一个语义层**。组件变量保留为「别名」，逐步指向语义令牌，实现渐进迁移、零破坏。

```
┌─────────────────────────────────────────────┐
│  第 1 层 · 原始令牌 (primitive)               │
│  纯数值，无语义。--blue-500 / --gray-100      │
│  --space-4 / --radius-md / --font-size-lg     │
└───────────────────┬─────────────────────────┘
                    │ 被引用
┌───────────────────▼─────────────────────────┐
│  第 2 层 · 语义令牌 (semantic)  ← 主题在这切换 │
│  --color-primary / --color-bg-default         │
│  --color-text-secondary / --color-border      │
│  html.light{} 和 html.dark{} 只覆盖这一层     │
└───────────────────┬─────────────────────────┘
                    │ 被映射
┌───────────────────▼─────────────────────────┐
│  第 3 层 · 组件别名 (component) ← 兼容旧代码   │
│  --header_background: var(--color-bg-elevated)│
│  --english_left_bg:   var(--color-bg-subtle)  │
│  旧变量名全部保留，只改其取值为语义令牌        │
└─────────────────────────────────────────────┘
```

**为什么这样能渐进迁移、不破坏**：
- 旧组件仍引用 `--header_background`，不用动模板；只是它的值改为指向 `--color-bg-elevated`。
- 昼夜切换仍只切换第 2 层（`html.light` / `html.dark`），第 3 层自动跟随——**现有 themePlugin 机制完全不变**。
- 新代码直接用第 2 层语义令牌，逐步淘汰第 3 层别名。

### 迁移优先级
1. **P0**：定义第 1+2 层（原始 + 语义令牌），集中写在 `light.css`/`dark.css` 顶部或新建 `tokens.css`。
2. **P0**：把第 3 层旧变量改为引用语义令牌（值替换，名不变）。
3. **P1**：把 `.vue` 中硬编码色全部替换为语义令牌。
4. **P1**：间距/字号/圆角逐步替换为尺度令牌。
5. **P2**：阴影 elevation、动效 motion 标准化。

> ⚠️ **具体取值（主色十六进制、字号阶梯数值、灰阶）依赖你在 [04-design-directions.md](./04-design-directions.md) 中选定的方向**。本文只定结构，不定最终色值。

## 五、预期收益

| 指标 | 现状 | 目标 |
|------|------|------|
| 互不相关强调色 | 17 个 | 1 主色 + 4 语义 + 灰阶 |
| 字号种类 | 15+ | 7–8 档尺度 |
| 改配色需触碰文件 | 4–5 处/色 | 1 处（语义层） |
| 硬编码色（.vue 内） | 多处 | 0 |
| 暗色一致性 | 部分缺失/对比不足 | 全量覆盖 + 对比达标 |
| 「品牌主色是什么」 | 答不出 | 一句话 |

# 实施路线图

> **前置条件**：本路线图在你 ① 复审通过 01–03 文档、② 选定 [04](./04-design-directions.md) 设计方向、③ 我回填 [03-design-spec.md](./03-design-spec.md) 具体色值后，方可启动。**本轮不执行任何一步代码。**
>
> 原则：先地基（令牌）后表层；先零风险（值替换、名不变）后重构；每阶段可独立交付 + 回归。

---

## 阶段 0 · 准备（选定方向后）

- [ ] HTML 高保真原型：选定方向的首页 + 文章页（眼见为实，定稿视觉）。
- [ ] 回填 03 号规范的「待定」色值、字体家族、灰阶。
- [ ] 新建 `src/assets/css/tokens.css`，在 `main.js` 中最先引入。

---

## 阶段 P0 · 设计令牌地基（零/低风险）

> 目标：把系统骨架立起来，**不改任何模板结构**，视觉基本不变或仅微调。

- [ ] **P0-1** 定义第 1 层原始令牌 + 第 2 层语义令牌（明/暗），写入 `tokens.css`。
- [ ] **P0-2** 把 `light.css`/`dark.css` 的旧组件变量改为**引用语义令牌**（值替换、名不变）→ 现有组件零改动自动跟随。
- [ ] **P0-3** 收敛品牌主色：17 个离散强调色 → 1 主色 + 语义色；逐一映射替换。
- [ ] **P0-4** 清除 `.vue` 内硬编码色：评论框 `green`、`content/InputBar` `#333ce1/#989cea`、Footer `#165dff/#adb4c5`、导航 `fill="#3170a7"`、口语教练 `skyblue` → 全改语义令牌。
- [ ] **回归**：昼夜切换、各页明/暗、首页 Hero 配图切换正常。

**风险**：低。**回归重点**：主题切换链（`themePlugin.js`）与 Hero 双图不受影响（只动第 2/3 层取值）。

---

## 阶段 P1 · 内容真实化 + 核心体验（中风险）

> 目标：从「半成品」到「真产品」，修复最影响体验的几处。

- [ ] **P1-1** Home 文章列表：移除硬编码 `v-for="movie in 10"` 假数据，抽 `ArticleCard` 组件接真实数据。
- [ ] **P1-2** WebNavigate / WebSite：`WebSite.vue` props 化（title/desc/url/favicon/category），接真实导航数据 + favicon + hover + 分区标题。
- [ ] **P1-3** About 页：填真实「关于」内容（自述/技能/时间线/社交/联系），重做布局 + 响应式，去硬编码。
- [ ] **P1-4** 文章页排版三件套：正文 `max-width: 70ch` + 行高 1.75 + 代码块明/暗配色统一；移除链接 `:before` emoji；TOC active 高亮。
- [ ] **P1-5** Test 页下线：移除生产路由或加 `import.meta.env.DEV` 守卫；修 `flex-wrap:1`→`flex:1`。
- [ ] **回归**：列表/导航/关于在明暗 + 移动端正常；文章长文阅读、代码块、TOC。

**风险**：中（涉及数据接入与组件抽取）。**回归重点**：数据为空/加载/失败时的表现（顺带补 P2 状态设计）。

---

## 阶段 P2 · 统一与精修（中风险，按方向落地）

> 目标：把选定方向的视觉语言铺满全站，统一组件与细节。

- [ ] **P2-1** 统一卡片：Home/WebNavigate/侧栏卡 hover 行为统一为上浮 + `--shadow-md`。
- [ ] **P2-2** 间距 / 字号 / 圆角全量替换为尺度令牌，淘汰非系统值。
- [ ] **P2-3** 阴影 elevation + 动效 motion 标准化；开屏门动画 1s → ≤500ms；主题切换加 `<html>` transition。
- [ ] **P2-4** AiEnglish 子应用按方向统一：气泡对比、`max-width: min(75%,680px)`、消息按钮不重叠、空/加载/错误三态、设置弹框分组、输入区令牌化。
- [ ] **P2-5** 主题偏好策略：首访按时段、手动切换后长期记忆；Hero 图按主题优先加载。
- [ ] **P2-6** 可访问性：`a { all: unset }` 修正、`:focus-visible` ring、图标按钮 `aria-label`、对比度复核、`prefers-reduced-motion`。
- [ ] **回归**：全站 5 页 + AiEnglish 在 明/暗 × 桌面/移动 共 4 组合下截图对比。

**风险**：中。**回归重点**：AiEnglish 改造面积大，需逐组件验证流式输出、语音、配置持久化不回归。

---

## 阶段 P3 · 增强（可选，低优先）

- [ ] 文章页：阅读进度条、复制代码按钮、图片 lightbox 增强。
- [ ] AiEnglish：消息搜索、`/` 快捷命令、时间戳。
- [ ] 导航页：分类筛选 / 搜索过滤。
- [ ] 性能：Hero 图压缩、字体子集化（中文字体按需）。

---

## 验收标准（每阶段通用）

1. `npm run lint` 零问题、`npm run build` 通过。
2. 明/暗 × 桌面/移动 4 组合视觉回归无异常。
3. **昼夜切换 + 首页配图切换功能完好**（受保护特性，每阶段必查）。
4. 无新增硬编码颜色 / 非系统间距（Code Review 检查）。
5. 关键页（首页/文章/AiEnglish）核心交互无回归。

---

## 工作量与顺序建议

```
方向选型 + 原型(阶段0)
   └─> P0 令牌地基          ← 必须最先，是后续一切的前提
         ├─> P1 内容真实化   ← 可与 P2 交叉
         └─> P2 统一精修     ← 视觉依赖方向定稿
               └─> P3 增强（随时插入，不阻塞）
```

> 建议以「阶段」为单位提交 PR + 复审，每个阶段独立可回滚，避免一次性大改难以审查。
